Travel manager B2B dashboard

UI/UX study case – Admin dashboard redesign for a travel management platform

Solution Expected Results

Overview

OK Roger is a B2B travel management platform for organizations. This is a conceptual study case created for design practice purposes, not an official project of OK Roger. The project goal was to redesign the information architecture and admin interface so that travel managers, finance leaders, and executives could more easily manage members, policies, and approvals.

My Role

Product Designer
Information Architecture, UI Design, UX, Dashboard Design

Problem

The previous admin experience lacked clarity and made it difficult to manage policies and approval flows. There was no strong visual hierarchy, and critical actions (like pending approvals) were hidden in complex menus. The challenge was to simplify and modernize the Information Architecture, while ensuring the final UI looked professional and beautiful.

Process

Deep dive
IA Redesign
UI Design
Prototype

Starting from the existing flows, I restructured the information architecture to clearly separate Policies, Approval Rules, and Pending Approvals. This clarity gave admins a mental model aligned with how they think and act.

Without user research (a constraint), I relied on UX reasoning and crafted a dashboard that prioritized critical actions: pending approvals, members without policy, outdated policies, and spend approaching limit.

Solution

The final design introduced a clear dashboard with quick actions and highlighted pending tasks. Policies and approval rules were separated into dedicated tabs, making it easier to scale and avoid duplication.

Policy creation was guided by a sticky preview footer acting as a checklist, reducing mistakes and building admin confidence.
